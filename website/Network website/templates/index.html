Welcome Network Attack Simulation for training Overview ▸

Introduction

Welcome to the Network Attack Simulation for training. This project provides an isolated environment for learning and performing common network attacks and at a later stage, you can learn how they can be detected and mitigated. By working through the labs, users will experience both the offensive and defensive sides of network security, gaining critical insight into how attackers operate and how defenders can protect their systems.

The labs simulate realistic network scenarios, allowing users to explore vulnerabilities, craft custom network packets, intercept communications, and implement IDS rules.

About This Project

This project focuses on teaching the fundamental concepts of network attack. It covers a series of common network attacks including:

  • Intro to Scapy and how to use Wireshark/Edgeshark on Docker containers
  • Reconnaissance and Service Enumeration using Nmap
  • ARP Cache Poisoning and Man-in-the-Middle (MITM) Attacks
  • Denial of Service (DoS) Attacks through SYN and TCP Flooding
  • TCP Reset Attack to Disrupt Active Connections
  • Reverse Shell Injection into Active Telnet Sessions

Users will utilize Scapy to craft custom packets and simulate attacks, and Wireshark to capture and analyze network traffic. Finally, defensive users will learn how to deploy Snort, a powerful Intrusion Detection System (IDS), to detect these attacks.

The labs are designed to provide a complete learning experience—from launching attacks, to analyzing network traffic, to detecting them using IDS, and finally learning how to defend against them.

There will also be a final “Challenges” page offering additional tasks for users who want to further practice their skills.

Learning Objectives

  • Understand the mechanics behind key network attacks at the protocol level.
  • Gain practical experience crafting packets and simulating attacks using Scapy.
  • Develop skills in network traffic analysis using Wireshark.
  • Learn how to detect common attacks using Snort and IDS rule creation.
  • Explore defense strategies such as static ARP entries, firewalls, and segmentation.
  • Understand the importance of proactive monitoring and defense in modern network security.

Lab Setup and Configuration

This section provides commands to set up your Docker containers for the security lab. Execute these commands in your terminal to start the required containers:

Docker Commands to Set Up Containers

docker run -dit --name victimA --network lab-net --ip 192.168.200.10 myvictimaimage:v2a
docker run -dit --name victimB --network lab-net --ip 192.168.200.11 victimbimage:v2b
docker run -dit --name attacker --network lab-net --ip 192.168.200.100 --privileged attackerimage:v2
docker run -dit --name hostattacker --network host --cap-add=NET_ADMIN --cap-add=NET_RAW hostattackerimage:v2

Network Setup Commands

Run these commands to set up the network and configure the containers with appropriate permissions and IPs. Ensure the Docker network 'lab-net' is created before running these commands:

Set Up Docker Containers with Specific IPs

docker network create --subnet=192.168.200.0/24 lab-net
docker run -dit --name victimA --network lab-net --ip 192.168.200.10 myvictimaimage:v2a
docker run -dit --name victimB --network lab-net --ip 192.168.200.11 victimbimage:v2b
docker run -dit --name attacker --network lab-net --ip 192.168.200.100 --privileged attackerimage:v2
