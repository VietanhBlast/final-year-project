Network Security Challenges

Welcome to the challenges page! This section contains various tasks designed to enhance your skills in network security. These challenges will help you apply the concepts you've learned from the labs to solve real-world security problems. Good luck!

Challenge 1: Ping of Death Attack ▸

Task Overview

In this challenge, you will exploit a vulnerability in the IP protocol by sending a maliciously crafted ping request, known as the "Ping of Death," to a victim machine. The objective is to cause the target system to crash or behave abnormally.

Challenge Requirements

  • Craft a ping request with a size greater than the maximum allowed size (e.g., 65,535 bytes).
  • Send this oversized ping to a victim machine (ensure the victim machine is configured to allow ICMP traffic).
  • Observe the impact of the attack on the victim machine.

Key Skills Tested

  • Crafting oversized ICMP packets
  • Exploit vulnerabilities in the IP protocol
  • Understanding the consequences of malformed packets
Challenge 2: Replay Attack ▸

Task Overview

In this challenge, you will perform a replay attack where previously captured valid packets are retransmitted to impersonate the sender. The goal is to simulate an attacker intercepting and reusing network traffic to gain unauthorized access or trigger an action.

Challenge Requirements

  • Capture a valid packet from a network session.
  • Replay the captured packet to perform an action like re-executing a command or replaying a transaction.
  • Monitor the response from the target system to verify the attack was successful.

Key Skills Tested

  • Packet capture and analysis
  • Packet replay and spoofing
  • Understanding how replay attacks can bypass authentication and other security mechanisms
Challenge 3: Modify Attacks to Avoid IDS Detection ▸

Task Overview

In this challenge, you will take existing attacks (e.g., SYN flood, ARP spoofing, etc.) and modify them to evade detection by an Intrusion Detection System (IDS), such as Snort. The goal is to understand how attack signatures can be altered to avoid triggering security systems.

Challenge Requirements

  • Select an existing attack (e.g., SYN flood, ARP spoofing, DDoS).
  • Modify your attack to avoid detection by Snort.
  • Test whether the modified attack triggers an alert in Snort.

Key Skills Tested

  • Understanding IDS signatures and detection techniques
  • Modifying attack traffic to evade IDS
  • Testing the effectiveness of IDS evasion
Challenge 4: Network Scan Detection Evasion ▸

Task Overview

In this challenge, you will use Nmap (use the codes in the Recon lab) to perform a network scan and then modify your scanning technique to bypass detection by Snort or another IDS. The objective is to understand how scan detection works and how attackers can evade IDS systems.

Challenge Requirements

  • Perform a basic Nmap scan on a target network.
  • Modify your Nmap scan to avoid detection by Snort, using options like timing, decoy scans, or fragmenting packets.
  • Observe if Snort detects the modified scan.

Key Skills Tested

  • Using Nmap for network scanning
  • Scan evasion techniques
  • Understanding how IDS can detect and block network reconnaissance

Challenge 4: Modify the Pre-made Attacks to Avoid IDS Detection

For this challenge, your goal is to modify the existing attacks to avoid detection by the IDS. To do this, focus on adjusting the timing and scan type, as well as using techniques like decoy scanning and scan stealth.

Hint: You can modify the Nmap command by changing the timing to -T1, using decoy scanning with -D, and experimenting with scan stealth techniques.
